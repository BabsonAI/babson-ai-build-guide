# Checks — prove it's production-ready

These are the **production gate**. If you're submitting a project for Babson hosting, **all three are mandatory**. If you're not hosting — just building or experimenting — you can ignore them.

They **verify**; the [guides](../guides) **explain**. Each check is terse and links back to the guide behind it.

Run them in order:

1. **[production-compliance-check.md](./production-compliance-check.md)** — is it hostable? (Toolkit, storage, data, secrets, repo, hand-off, shape.)
2. **[ai-patterns-review.md](./ai-patterns-review.md)** — are the AI parts built soundly? (Model calls, RAG, agents, evals, graceful degradation.)
3. **[submission-readiness-check.md](./submission-readiness-check.md)** — are you actually ready to hand it over? (The final gate.)

Point your AI tool at each file and your project, or run through them yourself.
